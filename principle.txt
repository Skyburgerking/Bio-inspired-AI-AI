# Gene Expression Decision Algorithm (GEDA) - Principles & Implementation
# 基因表达决策算法 - 原理与实现

## Core Principles | 核心原理

### 1. Five-Base Genetic System | 五进制碱基系统

**English Version:**
The GEDA system employs a novel five-base genetic encoding scheme, each representing distinct behavioral tendencies:

- **A (Aggressive Action)**: Represents proactive, risk-taking behavior focused on immediate gains and food-seeking
- **G (Stable Operation)**: Represents safe, pattern-maintaining behavior for consistent operation
- **C (Conservative Strategy)**: Represents cautious, risk-avoiding behavior for safety preservation
- **T (Flexible Adjustment)**: Represents adaptive, switching behavior for environmental changes
- **X (Mutational Exploration)**: Represents creative, random exploration for breakthrough strategies

**Key Features:**
- **Complementary Pairing**: A↔T, G↔C, X↔X (self-pairing mechanism)
- **Double-Strand Structure**: Each gene consists of complementary strands forming a decision "cipher" system
- **Dynamic Dominance**: Environmental pressure determines which strand becomes dominant

**中文版本:**
GEDA系统采用新颖的五进制遗传编码方案，每种碱基代表不同的行为倾向：

- **A (激进行动)**: 代表积极、冒险的行为，专注于即时收益和食物寻找
- **G (稳定操作)**: 代表安全、模式维持的行为，确保操作一致性
- **C (保守策略)**: 代表谨慎、规避风险的行为，保持安全
- **T (灵活调整)**: 代表适应性、切换行为，应对环境变化
- **X (突变探索)**: 代表创造性、随机探索，寻求突破性策略

**关键特性:**
- **互补配对**: A↔T, G↔C, X↔X (自我配对机制)
- **双链结构**: 每个基因由互补链组成，形成决策"暗号"系统
- **动态显性**: 环境压力决定哪条链成为显性链

### 2. Gene Expression Mechanism | 基因表达机制

**English Version:**

The gene expression process follows this workflow:

1. **Environmental Input**: Receive current state information from the environment
2. **Pressure Calculation**: Compute environmental pressure value (0.0-1.0)
3. **Dominant Strand Selection**: 
   - High pressure (>0.7): Activate aggressive strand (A/G bases dominant)
   - Low pressure (<0.3): Activate conservative strand (C/T bases dominant)
   - Medium pressure (0.3-0.7): Balanced strand activation
4. **Base Sequence Reading**: Read the selected strand's base sequence
5. **Complementary Verification**: Validate decision consistency through pairing
6. **Action Mapping**: Map verified sequence to executable actions
7. **Execution & Feedback**: Execute actions and collect environmental feedback
8. **Memory Update**: Update memory databases with learning outcomes

**Expression Logic:**
- High pressure environments favor aggressive/stable genes for immediate response
- Low pressure environments favor conservative/flexible genes for safe operation
- X-bases introduce exploration regardless of pressure level

**中文版本:**

基因表达过程遵循以下工作流程：

1. **环境输入**: 从环境接收当前状态信息
2. **压力计算**: 计算环境压力值(0.0-1.0)
3. **显性链选择**:
   - 高压(>0.7): 激活激进链(A/G碱基显性)
   - 低压(<0.3): 激活保守链(C/T碱基显性)
   - 中压(0.3-0.7): 平衡链激活
4. **碱基序列读取**: 读取所选链的碱基序列
5. **互补验证**: 通过配对验证决策一致性
6. **动作映射**: 将验证序列映射为可执行动作
7. **执行与反馈**: 执行动作并收集环境反馈
8. **记忆更新**: 用学习结果更新记忆数据库

**表达逻辑:**
- 高压环境偏好激进/稳定基因以快速响应
- 低压环境偏好保守/灵活基因以确保安全操作
- X碱基无论压力水平如何都引入探索行为

### 3. Mutation & Evolution | 变异与进化

**English Version:**

GEDA incorporates five mutation types simulating biological evolution:

1. **Insertion Mutation**: Adds 1-5 new bases randomly into the gene chain
2. **Replacement Mutation**: Replaces existing bases with new ones
3. **Deletion Mutation**: Removes bases from the gene chain
4. **Reordering Mutation**: Shuffles the order of bases within segments
5. **Complementary Mutation**: Switches bases with their complements (A↔T, G↔C)

**Evolutionary Characteristics:**
- **Memory Isolation**: New and old gene segments maintain separate memory databases
- **Natural Growth**: Gene chain length increases naturally over time
- **Evolutionary Strata**: Older segments form "fossil records" of past strategies
- **No Memory Pollution**: Segmented memory prevents cross-contamination

**中文版本:**

GEDA包含五种模拟生物进化的变异类型：

1. **插入变异**: 向基因链中随机插入1-5个新碱基
2. **替换变异**: 用新碱基替换现有碱基
3. **删除变异**: 从基因链中移除碱基
4. **重排变异**: 打乱基因段内碱基的顺序
5. **互补变异**: 将碱基与其互补碱基交换(A↔T, G↔C)

**进化特性:**
- **记忆隔离**: 新旧基因段保持独立的记忆数据库
- **自然生长**: 基因链长度随时间自然增长
- **进化地层**: 老基因段形成过去策略的"化石记录"
- **无记忆污染**: 分段记忆防止交叉污染

The system demonstrates contextual intelligence through:

**Feature Extraction Process:**

5. Select expression scheme with highest confidence

**Feature Dimensions:**
- Food proximity and direction
- Obstacle density and distribution
- Available space and movement freedom
- Historical decision success patterns
- Current environmental pressure level

系统通过以下方式展现情境识别智能：

3. 查询每个基因段的历史成功率
4. 生成结合匹配度和历史的置信度分数
5. 选择置信度最高的表达方案

**特征维度:**

### 2. Progressive Learning Ability | 渐进学习能力

**English Version:**

The system learns progressively through three mechanisms:

**Success Reinforcement:**
- Effective decisions increase expression frequency
- Related gene segments activate collaboratively
- Success patterns become more dominant over time

**Failure Silencing (Methylation Analogy):**
- Unsuccessful decisions receive methylation markers
- Marked segments are temporarily suppressed
- Genetic information remains preserved for potential reactivation

**Mutational Innovation:**
- X-bases introduce random exploration
- May discover novel, successful strategies
- Provides escape from local optima

**中文版本:**

系统通过三种机制进行渐进学习：

**成功强化:**
- 有效决策增加表达频率
- 相关基因段协同激活
- 成功模式随时间变得更加显性

**失败沉默(甲基化类比):**
- 不成功决策接收甲基化标记
- 标记的基因段被暂时抑制
- 遗传信息保留以备潜在重新激活

**突变创新:**
- X碱基引入随机探索
- 可能发现新颖的成功策略
- 提供逃离局部最优的途径

**English Version:**

GEDA exhibits adaptive evolution through:

**Environmental Adaptation:**

## Implementation Design | 实现设计

### 1. Lightweight Architecture | 轻量级架构

**English Version:**

Core implementation structure:

```
class GEDAAgent:
    def __init__(self):
        self.gene_chain = []          # Genetic chain storage
        self.memory_dbs = {}          # Segmented memory databases
        self.expression_cache = {}    # Expression result caching
        self.environment_history = deque(maxlen=100)  # Recent environment records
    
    def perceive_and_act(self, state):
        # 1. Extract environmental feature vector
        features = extract_features(state)
        
        # 2. Activate matching gene segments
        active_segments = self.activate_segments(features)
        
        # 3. Generate decision action
        action = self.express_decision(active_segments, features)
        
        # 4. Execute and learn
        result = self.execute_and_learn(action, state)
        
        return action, result
```

### 3. Performance Optimization | 性能优化

**English Version:**

Optimization strategies:

**Caching Mechanism:**
- High-frequency expression results are cached
- Reduces repetitive computation for similar situations
- Cache invalidation based on environmental changes

**Segment Loading:**
- Gene segments activated on demand
- Reduces memory footprint
- Improves response time for frequent patterns

**Parallel Processing:**
- Multiple gene segments can be evaluated concurrently
- Speeds up decision-making in complex environments
- Enables consideration of multiple strategies simultaneously

**中文版本:**

优化策略：

**缓存机制:**
- 高频表达结果被缓存
- 减少类似情况的重复计算
- 基于环境变化的缓存失效

**分段加载:**
- 按需激活基因段
- 减少内存占用
- 提高频繁模式的响应时间

**并行处理:**
- 可以并发评估多个基因段
- 加速复杂环境中的决策
- 允许同时考虑多种策略
